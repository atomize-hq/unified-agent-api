- **Name**: Universal fork semantics via `AgentWrapperRunRequest.extensions["agent_api.session.fork.v1"]`
- **Goal / value**: Let orchestrators fork a new session/thread from `"last"` or `"id"` across built-in backends with deterministic, fail-closed validation, pinned selection-failure behavior, and non-interactive safety.
- **Type**: risk (capability seam with a high-risk Codex mapping)
- **Scope**

      - deterministic mapping per backend-owned mapping contracts,
      - selection-failure translation to pinned safe messages (`"no session found"` / `"session not found"`) plus the terminal `Error` event rule when a stream exists (`extensions-spec.md`),
      - mutual exclusivity with `agent_api.session.resume.v1` enforced as `InvalidRequest` **only when both keys are supported** (R0 precedence is authoritative),
      - non-interactive safety (`agent_api.exec.non_interactive`, default `true`) mapped per backend contracts (no hangs on approvals/permissions prompts).
