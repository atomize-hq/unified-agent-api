# Unified Agent API (Canonical Specs)

This directory contains the **canonical, normative** Unified Agent API specifications.

Source ADR:
- `docs/adr/0009-unified-agent-api.md`

Canonical specs:
- `docs/specs/unified-agent-api/contract.md`
- `docs/specs/unified-agent-api/run-protocol-spec.md`
- `docs/specs/unified-agent-api/event-envelope-schema-spec.md`
- `docs/specs/unified-agent-api/capabilities-schema-spec.md`
- `docs/specs/unified-agent-api/extensions-spec.md`
- `docs/specs/unified-agent-api/support-matrix.md`

Draft specs:
- `docs/specs/unified-agent-api/mcp-management-spec.md`

Generated artifact:
- `docs/specs/unified-agent-api/capability-matrix.md` (generated by `cargo run -p xtask -- capability-matrix`)
- Any change that adds or removes a published capability id for a lifecycle-eligible agent MUST
  regenerate that artifact in the same change so the lifecycle-backed publication surface stays
  aligned with committed publication truth.

The support publication contract lives in `docs/specs/unified-agent-api/support-matrix.md`. The
generated `capability-matrix.md` remains a separate lifecycle-backed publication surface and does
not define support policy.

Planning/execution artifacts remain in:
- the Unified Agent API planning/provenance area under `.archived/project_management/next/`
